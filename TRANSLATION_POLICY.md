# Translation Policy

## Default direction
- Source: English
- Target: Korean

## Priority (strict)
1) Technical meaning accuracy (highest)
2) Terminology consistency (use glossary first)
3) Natural Korean readability

## Target reader / tone
- Reader: non-expert who still wants precise technical meaning
- Korean honorifics (존댓말)
- Short, decisive sentences
- Avoid excessive explanation unless absolutely necessary to preserve meaning

## Formatting / structure
- Preserve Markdown structure, headings, lists, tables
- Preserve links, filenames, inline code, and any quoted identifiers
- Do not translate code blocks (``` ... ```)
- Keep units, numbers, versions exactly as written

## Notes / translator’s notes
- If context is missing, add minimal note in-line:
  - [역주: ...]
- Never add new facts; never remove constraints
